## Key Conclusions from the AST Scoring

### 1. The Industry Has Solved Compute Isolation — But Almost Nothing Else

**L1 (Compute) and L3 (Filesystem) are universally strong.** Every single product scores S≥2 on both. 9/22 products hit S:4 (microVM/hardware isolation) on L1. The industry has clearly converged on "put the agent in a box" as table stakes.

**But the upper layers are a wasteland:**
- **L5 (Credentials):** 11/22 products (50%) score S≤1 — meaning half the market has credentials directly accessible to the agent. This is the single weakest layer across the entire landscape.
- **L6 (Governance):** 8/22 products have S≤1 or don't address it at all (6 nulls). Only 3 products reach S:3. Zero reach S:4.
- **L7 (Observability):** Only 1 product (nono) reaches S:3. Zero reach S:4. The industry has almost no tamper-evident audit capability.

### 2. The Exfiltration Gap Is the #1 Systemic Risk

11 out of 22 products have a weak L4 or L5 (or both), creating a direct data exfiltration path. The pattern is clear: products build a beautiful microVM (L1:4) then hand the agent raw API keys as env vars (L5:1) with unrestricted network (L4:0-1). The README nails it: **"the most common configuration is the most dangerous one."**

Notable offenders:
- **Devin**: L1:4 but L4:0 + L5:1 — a fortress with the front door open
- **Cursor**: L1:3 but L4:1 + L5:0 — cooperative network gate + no credential management
- **packnplay**: L4:0 + L5:1 + L6:0 + L7:0 — basically a convenience wrapper, not a security tool

### 3. Only 6 Products Achieve Full Threat Coverage (7/7)

**Google Agent Sandbox, Claude Code (web), Copilot coding agent, Replit, Deno Sandbox, and Deno Deploy.** All share one trait: S≥2 across every primary defense layer. Notably:
- 5 of 6 are **cloud-managed** — it's much easier to achieve full coverage when you control the entire stack
- Google Agent Sandbox is the only one with L6:2+ via a real policy engine (ADK callbacks + Model Armor)
- Deno Sandbox has the strongest overall profile: Firecracker L1:4 + credential proxying L5:3

### 4. Cloud-Managed Dominates, Local-First Is Underserved

13/22 products are cloud-managed. Only 7 support Linux locally, 7 macOS. The local-first space has two standouts:
- **nono**: S:3 across 6 of 7 layers with kernel-enforced irreversible restrictions + cryptographic audit. Highest per-layer scores of any local tool. But it's "early alpha."
- **Codex CLI**: Solid S:3 on L1/L3/L4 with Landlock+seccomp, but L2:0 (no resource limits)

### 5. Governance (L6) Is the Frontier

L6 is where the market is least mature. 6 products don't address it at all. Only 3 reach S:3 (nono, Replit, Deno Sandbox). No product reaches S:4. This makes sense — governance requires understanding *intent*, which is fundamentally harder than enforcing *boundaries*. The products that do L6 well (Leash, Warden, Google Agent Sandbox) use policy engines (Cedar, OPA, ADK callbacks), but these are all software-enforced and bypassable.

### 6. Composition Is Not Optional — It's the Architecture

The fingerprints make it obvious: no product fills all 7 positions well. The taxonomy explicitly designs for this with the "take the max at each layer" stacking rule. The clearest composition patterns:
- **E2B (4/4/4/2/1/-/1) + Warden (2/0/2/2/2/2/2) = 4/4/4/2/2/2/2** — gaps filled
- **Claude Code local (3/-/3/3/1/2/2) + nono (3/-/3/3/3/3/3) = 3/-/3/3/3/3/3** — only L2 uncovered

### 7. Evidence Quality Is Uniformly Low

Every single product has `evidence_level: docs` and `last_reviewed: null`. None have been verified through hands-on testing or source code review. The scores are based on marketing docs and official documentation — which means the *actual* security posture could be worse than reported. The Blaxel entry explicitly calls out a "blog-to-docs gap on security features."

**TL;DR:** The sandbox market has converged on strong compute isolation (L1-L3) but is systematically weak on credentials (L5), governance (L6), and observability (L7). Half the products have a direct exfiltration path. Full coverage requires composing 2+ products. The local-first space is underserved. And nobody has verified any of these scores yet.
